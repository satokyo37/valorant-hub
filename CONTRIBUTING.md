# Contributing Guide

VALORANT HUB に興味を持っていただきありがとうございます。  
このドキュメントは、**開発・拡張を行う人向け**に  
プロジェクトの方針や構成、開発時の注意点をまとめたものです。

---

## 🧠 プロジェクトの方針

- シンプルであることを最優先
- 過度な設計・抽象化をしない
- 小さく作って、必要になったら広げる
- 「消しやすさ」「変更しやすさ」を重視する

このプロジェクトは **実験的・学習目的の個人開発**です。  
商用サービスレベルの厳密さは求めていません。

---

## 🧩 現在の構成思想

- フロントエンドのみで完結
- 認証なし
- データベースなし
- バックエンドなし

まずは **クライアントサイドだけで成立する機能**を優先します。  
必要になった段階で、API やバックエンドを検討します。

---

## 📁 ディレクトリ構成の考え方

```text
src/
  app/          # ページ・画面（Next.js App Router）
  data/         # 静的データ（エージェント情報など）
  components/   # 再利用可能なUIコンポーネント
  lib/          # ユーティリティ・共通ロジック
public/
  agents/       # エージェント画像などの静的アセット
```

- 機能追加は基本的に src/app/ 配下に閉じる
- グローバルに広がる実装はできるだけ避ける

---

## 📦 開発環境について
パッケージマネージャー
- npm を使用しています

```bash
npm install
npm run dev
```

---

## 🗂 パスエイリアス

以下のパスエイリアスを使用しています。

```ts
@/* -> src/*
```

設定場所：
- `tsconfig.json`

使用例：

```ts
import { AGENTS } from "@/data/agents";
```

---

## 🖼 アセット（画像）に関する方針

- エージェント画像は public/agents/ 配下に配置
- 外部URLへの直リンク（ホットリンク）は行わない
- ファイル名はエージェントの id に合わせる

例：

```text
public/agents/jett.png
public/agents/omen.png
```

---

## 🎨 UI / UX に関する方針

- 見やすさ・分かりやすさを最優先
- アニメーションは控えめに
- Result（結果表示）はリッチにしてもよい
- 一覧・設定系UIは軽量に保つ
- モバイル表示を常に意識する

---

## 🔧 機能追加のガイドライン

- 新機能は「まず小さく」
- クライアントサイドで完結できるかを最初に考える
- 状態管理は React の基本機能で足りるか検討する
- 将来の削除・変更を想定した実装にする

---

## 📝 コミットメッセージの目安

厳密なルールはありませんが、以下を推奨します。
- feat: 新機能
- fix: バグ修正
- chore: 設定変更・雑務
- docs: ドキュメント更新

例：

```text
feat: add agent roulette animation
docs: update contributing guide
```
